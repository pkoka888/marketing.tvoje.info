# Product Planning Reference Guide

## Product Vision Framework

### Vision Statement Template

```markdown
## Vision: [Product Name]

For [target customer segment]
Who [statement of the problem or need]
The [product name] is a [product category]
That [key benefit, compelling differentiation]
Unlike [primary competitive alternative]
Our product [unique value proposition]
```

## Product Strategy Elements

### Strategic Pillars

| Pillar | Description | Example |
|--------|-------------|---------|
| Vision | Long-term north star | Become the leading DevOps platform |
| Mission | How we achieve vision | Empower teams to ship faster |
| Values | Guiding principles | User-first, security, simplicity |
| Goals | Measurable objectives | 2x revenue, 90% retention |

```markdown
## Q1 OKRs
```

## Prioritization Frameworks

### RICE Scoring

```markdown
## RICE Prioritization

**RICE Score = (Reach Ã— Impact Ã— Confidence) / Effort**

| Feature | Reach | Impact | Confidence | Effort | RICE Score |
|---------|-------|--------|------------|--------|------------|
| Feature A | 10000 | 3 | 80% | 10 | 240 |
| Feature B | 5000 | 5 | 90% | 20 | 112.5 |
```

**Scales:**
- **Reach:** Users affected per quarter (1-100000)
- **Impact:** 0.25 (minimal) to 3 (massive)
- **Confidence:** 50% to 100%
- **Effort:** Person-months (1-100)

### MoSCoW Prioritization

| Category | Description | Target |
|----------|-------------|--------|
| Must Have | Essential for release | 60% of features |
| Should Have | Important but not critical | 20% of features |
| Could Have | Desirable if time permits | 15% of features |
| Won't Have | Not this release | 5% of features |
