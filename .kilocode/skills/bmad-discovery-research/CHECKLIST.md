# Discovery Research Checklist

## Phase 1: Define Research Scope

- [ ] **1.1** Articulate the research question or objective
- [ ] **1.2** Define scope (what's in/out of scope)
- [ ] **1.3** Identify key stakeholders and decision-makers
- [ ] **1.4** Establish success criteria and desired outcomes
- [ ] **1.5** Define timeline and resource constraints
- [ ] **1.6** Identify required expertise and data sources

## Phase 2: Conduct Primary Research

- [ ] **2.1** Interview internal stakeholders and domain experts
- [ ] **2.2** Survey target users or customers if applicable
- [ ] **2.3** Analyze existing data (analytics, support tickets, feedback)
- [ ] **2.4** Conduct user observations or contextual inquiry
- [ ] **2.5** Gather competitive products for hands-on analysis
- [ ] **2.6** Document findings in structured format

## Phase 3: Conduct Secondary Research

- [ ] **3.1** Search academic and industry publications
- [ ] **3.2** Analyze competitor websites, documentation, and blogs
- [ ] **3.3** Review market reports and analyst perspectives
- [ ] **3.4** Study technology documentation and RFCs
- [ ] **3.5** Examine open source projects and case studies
- [ ] **3.6** Gather industry benchmarks and metrics

## Phase 4: Competitor Analysis

- [ ] **4.1** Identify primary and secondary competitors
- [ ] **4.2** Map competitor
- [ ] features and capabilities **4.3** Analyze pricing and positioning strategies
- [ ] **4.4** Evaluate user experience and design patterns
- [ ] **4.5** Assess technical architecture and stack choices
- [ ] **4.6** Identify strengths, weaknesses, opportunities, threats

## Phase 5: Technology Evaluation

- [ ] **5.1** Define evaluation criteria and weighting
- [ ] **5.2** Research candidate technologies or tools
- [ ] **5.3** Assess maturity, community, and ecosystem
- [ ] **5.4** Evaluate licensing, cost, and vendor lock-in
- [ ] **5.5** Consider integration complexity and dependencies
- [ ] **5.6** Create comparison matrix with trade-offs

## Phase 6: Synthesis and Analysis

- [ ] **6.1** Organize findings by theme or category
- [ ] **6.2** Identify patterns, trends, and insights
- [ ] **6.3** Validate findings against multiple sources
- [ ] **6.4** Quantify results where possible (metrics, percentages)
- [ ] **6.5** Document conflicting information and perspectives
- [ ] **6.6** Highlight gaps and areas needing further research

## Phase 7: Recommendations

- [ ] **7.1** Develop actionable recommendations
- [ ] **7.2** Prioritize recommendations (must-have, nice-to-have)
- [ ] **7.3** Document trade-offs and decision criteria
- [ ] **7.4** Outline implementation considerations
- [ ] **7.5** Identify risks and mitigation strategies
- [ ] **7.6** Suggest next steps and validation methods

## Phase 8: Documentation

- [ ] **8.1** Write executive summary (1-2 pages)
- [ ] **8.2** Document methodology and sources
- [ ] **8.3** Present findings with evidence and citations
- [ ] **8.4** Create visual aids (charts, diagrams, matrices)
- [ ] **8.5** Compile appendices with detailed data
- [ ] **8.6** Review for accuracy and completeness

## Quality Gates

- [ ] Research questions are specific and answerable
- [ ] Multiple sources consulted for key findings
- [ ] Competitor analysis covers market leaders and challengers
- [ ] Technology evaluation uses weighted criteria
- [ ] Recommendations are evidence-based with trade-offs
- [ ] Limitations and uncertainties are documented
- [ ] Sources are cited and verifiable
