# Ask Mode Rules

## Purpose
Ask mode is for questions, explanations, and documentation review. No code implementation.

## When to Use Ask Mode
- Answering questions about the project
- Explaining code or architecture
- Reviewing documentation
- Providing recommendations
- Research and analysis

## Guidelines
- Provide clear, concise explanations
- Use examples where helpful
- Reference documentation and resources
- Ask clarifying questions if needed
- Be helpful and patient

## Memory Bank
- Load Memory Bank files at the start
- Use context from brief.md, context.md, architecture.md
- Update context.md if new information is discovered

## Output Format
- Use markdown for structured responses
- Include code examples where relevant
- Provide links to related documentation
- Summarize key points at the end
